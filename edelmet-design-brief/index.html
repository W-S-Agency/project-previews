04 · 14
04 · Паттерны B2B-Mittelstand DE

Визуальные конвенции немецкого B2B-сегмента

✅ Что работает

  • Trust-Bar above the fold: сертификаты (EfbV, DIN, ISO, TÜV) сразу видны
  • Чистая типографика: Inter / IBM Plex / SF Pro · никаких декоративных шрифтов
  • Нейтральные палитры: белый/серый base + 1 corporate accent (синий/зелёный)
  • Структурированная информация: таблицы, чек-листы, процессы 1-2-3
  • Real photography: фуры, склад, сотрудники, оборудование (НЕ stock-iconography)
  • Контактные данные сразу: phone, email, физический адрес в шапке/футере
  • Documenten-Center: downloads (PDF certificates, AGB, datasheets)

❌ Чего избегаем

  • Игривые иллюстрации в стиле «startup landing page» — для B2B-mittelstand читается как несерьёзно
  • Большие hero-видео с маркетинговым stock-footage без content
  • Эмодзи в навигации и заголовках страниц
  • 3D-renders / abstrakte Visuals без связи с реальностью бизнеса
  • Большие gradients (Y-Combinator-style) — выглядит несолидно для compliance-тематики
  • Ankauf-Shop UX: «Verkaufe deinen Laptop»-кнопки, цены на главной
Главное правило: «Sachlich, präzise, vertrauenswürdig» — это не «boring». Это эстетика подтверждённой компетенции. Compliance-Officer / IT-Leiter платят за уверенность, не за wow-эффект.

10 · Design-Kit Scope

Что должно быть в твоём deliverable

Colors

  • • Primary (corporate)
  • • Secondary (accent)
  • • Neutrals (5-7 оттенков)
  • • Semantic: success / warning / error / info
  • • Hover / active / disabled states

Typography

  • • Display (H1, hero)
  • • Headings (H2-H4)
  • • Body (large / regular / small)
  • • UI (labels, captions)
  • • Mono (для tabular data, AVV-codes)
Желательно variable font, поддерживающий umlaute

Components

  • • Buttons (primary, secondary, tertiary, ghost)
  • • Forms (inputs, selects, file upload, multi-step)
  • • Cards (service, ZG, case)
  • • Tables (AVV codes, certificates)
  • • Navigation (header, footer, breadcrumbs)
  • • Modals, tooltips, badges, pills

Spacing & Grid

  • • 8px-scale (4-8-16-24-32-40-48-64-80)
  • • Container max-width
  • • Breakpoints: mobile / tablet / desktop / wide
  • • Section padding standards

Iconography & Imagery

  • • Icon set (consistent, outline или filled)
  • • Photography style guide (real / industrial)
  • • Illustration style (если применимо — minimal)
  • • Certificate/badge treatment

Patterns / Mock-Pages

  • • Hero variants (3-4)
  • • Trust-Bar variants
  • • Service-Grid layout
  • • Form layout (5-stufiges B2B-formular)
  • • Footer (corporate, dense)
⚠️ Логотип EDELMET существует — не редизайн логотипа. Только treatment в системе (clear-space, на различных фонах, lockups).

12 · Deliverables

Что отдаёшь по итогу

Figma-файл со структурой:

  • 📁 Foundations — colors, typography, spacing, shadows, radii
  • 📁 Components — buttons, forms, cards, tables, navigation, modals
  • 📁 Patterns — Hero / Trust-Bar / Service-Grid / Process / Footer
  • 📁 Mock-Pages (минимум 3): Startseite · ZG-LP-template · Service-Page
  • 📁 States — hover, active, disabled, error для всех components

Технические требования

  • → Mobile-first: 360 / 768 / 1280 / 1440 breakpoints
  • → Auto-Layout для всех components — для разработчика на Odoo
  • → Variants для всех states (Figma component variants)
  • → Variables для colors / typography (light + поддержка dark в будущем)
  • → Доступность: WCAG 2.1 AA contrast minimum
